# Отчет по лабораторной работе №2

## Тема: "Методы сортировки и поиска"

**Вариант:** №6-4 (Метод: Двоичная вставка; Таблица: Строковый ключ, хранение вместе, Минимальное число элементов таблицы: 14)

---

### 1. Описание задачи и варианта

Целью работы является реализация системы управления таблицей данных с использованием динамической памяти.

* **Метод сортировки:** Сортировка двоичными вставками (Binary Insertion Sort). Оптимизация метода простых вставок за счет поиска позиции элемента с помощью бинарного поиска ($O(\log n)$ сравнений вместо $O(n)$).
* **Структура данных:** Таблица с записями типа `Cell`, содержащими строковый ключ (до 100 символов) и 64-битное целое число (`int64_t`). Хранение ключа и данных осуществляется "вместе" в одной структуре.
* **Управление памятью:** Использование динамического массива с автоматическим расширением (`realloc`) при заполнении.

---

### 2. Алгоритмическая сложность

#### Сортировка двоичными вставками

1. **Сравнения (Comparisons):** Благодаря бинарному поиску, количество сравнений составляет **$O(n \log n)$** во всех случаях.
2. **Пересылки (Moves):** Поскольку элементы хранятся в массиве, для вставки требуется физический сдвиг "хвоста" вправо. Количество пересылок остается **$O(n^2)$**.
3. **Итог:** Общая временная сложность — **$O(n^2)$**, однако на практике метод работает быстрее простых вставок за счет уменьшения числа вызовов `strcmp`.

#### Двоичный поиск

* Временная сложность: **$O(\log n)$**.
* Требование: Работает только на предварительно отсортированном массиве.

### 3. Структура проекта

Программа разделена на три модуля для обеспечения инкапсуляции и модульности:

1. **`algoritm.h`**: Объявления структур `Cell`, `Arr`, прототипы функций сортировки и поиска. Использует `#pragma once` для защиты от повторного включения.
2. **`algoritm.c`**: Реализация алгоритмов бинарного поиска места вставки, сортировки и очистки памяти.
3. **`main.c`**: Точка входа, логика интерактивного или пакетного ввода данных, управление памятью и тестирование.

---

### 4. Инструкция по сборке и запуску (UNIX)

Для сборки проекта используется утилита `make`.

#### Команды терминала

1. **Сборка программы:**

    ```bash
    make all
    ```

    *Компиляция выполняется с флагами `-Wall -Wextra -g -std=c11` для обеспечения безопасности и возможности отладки.*

2. **Запуск в автоматическом режиме (тестирование файлом):**

    ```bash
    ./prog.bin < examples/random.txt
    ```

    *Программа считает данные из файла, выведет исходный массив, отсортирует его и выполнит поиск ключей, указанных в конце файла.*

3. **Очистка проекта:**

    ```bash
    make clean
    ```

---

### 5. Примеры тестовых случаев

Для проверки корректности реализованы три сценария (файлы прилагаются):

1. **`ordered.txt`**: Данные уже отсортированы (A-Z). Проверка "лучшего случая".
2. **`reverse.txt`**: Данные в обратном порядке (Z-A). Проверка самого трудоемкого случая для сдвига элементов.
3. **`random.txt`**: Случайный набор слов (фрагмент текста). Проверка корректности работы бинарного поиска в реальных условиях.

**Формат входных данных:**
`(ключ число)`
*Например:* `(apple 10) (banana 20)`

---

### 6. Заключение

В ходе работы была реализована устойчивая (stable) сортировка. Применение `realloc` с коэффициентом расширения 2 позволило достичь амортизированной сложности $O(1)$ для добавления элементов. Программа корректно обрабатывает пустые вводы и ошибки выделения памяти.
